# Archive Directory

This directory contains historical design documents, implementation plans, and code reviews that have been completed or are no longer actively referenced.

## Structure

### `implemented/`
Design documents and implementation plans that have been successfully implemented in the codebase:

- **Protocol Registry** - Protocol registration framework (implemented in `internal/protocol/registry/`)
- **UDP Handshake** - SYN/ACK handshake mechanism (implemented in `internal/protocol/udp/`)
- **HTTP Poll Fragment Analysis** - Fragment handling for HTTP Poll protocol
- **Firewall Bypass Analysis** - HTTP Poll protocol implementation
- **Code Quality Improvements** - Various code quality fixes that have been applied
- **Naming Consistency** - Naming standardization work
- **Responsibility Overlap** - Architecture refactoring work
- **UDP Design** - Initial UDP protocol design

### `code-reviews/`
Historical code review documents and analysis:

- Comprehensive code reviews from 2024-2025
- Fix plans derived from code reviews
- External reviews (ChatGPT, etc.)

## Note

These documents are kept for historical reference and to document the evolution of the codebase architecture. They should not be considered active implementation plans.

For current architecture and implementation guides, refer to the main `docs/` directory.
